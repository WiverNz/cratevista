# Name and positioning

## Recommended name

# CrateVista

Public title:

> **CrateVista — Interactive Rust Architecture & Documentation Explorer**

Tagline:

> Turn any Rust workspace into an interactive architecture map.

## Package identity

```text
GitHub repository: cratevista
crates.io package: cargo-cratevista
binary: cargo-cratevista
Cargo command: cargo cratevista
```

## Primary searchable phrases

Use naturally in the README, website, release notes, and demo descriptions:

- Rust architecture explorer
- interactive Rust documentation
- visualize Rust workspace
- Cargo dependency graph
- rustdoc JSON viewer
- Rust module graph
- Rust trait implementation graph
- Rust codebase visualization
- Cargo subcommand documentation generator

## Differentiation

CrateVista should be described as:

- broader than a Cargo dependency graph;
- more interactive than generated static SVG;
- architecture-oriented rather than a replacement for rustdoc API pages;
- browser-based rather than terminal-only;
- local-first and source-private;
- capable of mixing discovered Rust structure with manually curated runtime flows.

## Naming cautions

Avoid using these as the primary product name:

- `RustViz`: already associated with existing Rust visualization tooling.
- `RustLens`: already used by a Rust code inspection project.
- `FerrisScope`: already used by another Rust project.
- `Rust Atlas`: broad and ambiguous, with unrelated uses.

A final availability and trademark check is required immediately before public release.
